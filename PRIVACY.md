# Privacy Policy 🔒

Little Alchemist 2 Cheats is a free, fan-made recipe lookup tool. It is built to be useful without collecting personal data, selling data, or showing ads.

## What We Collect 🧪

The app does not ask for an account, name, email address, payment details, or any other personal profile information.

The app may use browser storage for small local preferences and app state:

- Discovered combination checkmarks.
- Install prompt dismissal state.
- PWA cache data needed to make the app work offline.

This information is stored on your device by your browser. It is not sent to this project, a backend server, or a maintainer inbox.

## Analytics 📊

The app does not include Google Analytics or any other analytics tracker.

This project does not use analytics to build user profiles, sell audiences, or target advertising.

## Ads And Payments 🚫

This project is free to use.

There are no ads, sponsored placements, affiliate links, paywalls, subscriptions, or paid unlocks. The goal is to keep the recipe finder fast, useful, and low-friction.

## Offline Data 📲

The PWA service worker stores app files, recipe data, the logo, and element icons locally so the app can continue working offline.

These cached files are static app assets. They are not a record of your browsing history or discovered combinations.

## Third-Party Services 🌐

The live site may rely on:

- GitHub Pages to host the app.

The app should not need to fetch Little Alchemy image assets from external image hosts at runtime because those assets are stored locally in `public/`.

## Clearing Local Data 🧹

You can clear discovered combination checkmarks from inside the app using the clear discovered combinations action.

You can also remove all local app data through your browser settings by clearing site data for the app domain. That may also remove PWA cache files and install prompt preferences.

## Contact 💬

For privacy concerns, security concerns, or questions about this policy, please open an issue using the guidance in [SUPPORT.md](SUPPORT.md) or report sensitive issues through [SECURITY.md](SECURITY.md).
